Data types

$ - global variables.

0004: $MyVar = 100

@ before an identifier: a label. The identifier is used to reference the address in the script 

0002: jump @MyLabel

@ after a number: a local variable.

0006: 100@ = 10

'...' - a fixed-length string containing up to 7 characters (or 15 in San Andreas). 

03A4: name_thread 'MAIN'

An empty string is also allowed: ''

The single quote character needs to be escaped with the backward slash. 

03A4: name_thread 'scr\'ipt'

"..." - a variable-long string (the maximum length is determined by the opcode)

An empty string is allowed: ""

The double quote character needs to be escaped with the backward slash. 

0662: write_debug_message "Hello, \"world\"! \n 'Here we go!'"

s$ - a global string variable

05A9: s$MyString = 'GLOBAL'

@s - a local string variable

05AA: 1@s = 'LOCAL'

v$ - a global long string variable 

06D1: v$MyString = "LONG_GLOBAL"

@v - a local long string variable 

06D2: 1@v = "LONG_LOCAL"

# - a model identifier from '.ide' files 

0247: request_model #CELLPHONE

& - ADMA (Advanced Direct Memory Access).
Reads/writes the values within the SCM even not in the variables block.
Does not affect the second segment size.

ADMA can be used the same as a global variable.

&57 += &120(&231,4i)

0x - a hexadimal number
-0x - a negative hexadimal number 

0004: $var = -0xBB08

The number has to be within the -0x80000000..0x7FFFFFFF range.
