Sanny Builder Error Messages



Model name can not be used

This error is happened when you try to use a model name in an SCM file without header, and the compiler does not know what numeric ID this model has. Most often this happens when you compile a CLEO script with a model name that was defined in the original main.scm in the DEFINE OBJECT list (for example, #KEYCARD). These #names unlike those ones defined in the vehicles.ide, peds.ide default.ide can not be compiled.

The most notably effect of this error is that in the game you will see an INFO pickup (#INFO) instead of expected model.

Possible solutions: use a numeric identifier (ID) instead of the model name. For example, for the #KEYCARD ID = 1581. You may find out needed ID in the game IDE files. Also you can use a a SB built-in function of converting the models names to their ID and vice versa (Ctrl+Alt+H).
