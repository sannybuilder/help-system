  Console

The console is intented to switch the advanced options of Sanny Builder. Mostly they relate to the decompiler features and you can see the changes when decompile a file. The console window is called by pressing Ctrl + ~. You may write the commands and the parameters in the edit line. After typing a command press Enter to process. The console will display a result.

All commands are case-insensitive.

The available commands are:

help � shows the help information about the console

clear � clears the console

list � shows in the console the list of all commands

The parameter (optional):

list options � after input list options the console will display all possible options to toggle.

toggle � the major command to enable or disable the program options

The parameter:

toggle <option name> [on/off]

The options list can be different in other version. To get all possible options, type list options.

The command line might be ended either with word ON (the option will be enabled) or word OFF (the option will be disabled), or nothing (the option will be toggled from ON to OFF or from OFF to ON).

Note: First of all the debug mode should be enabled with command toggle debug_mode

toggle CODE_OFFSETS the decompiler will print offsets of each opcode
toggle VAR_COUNTER after compiling the console will contain the list of the global variables used once in the code. It is very useful to search the useless variables.

toggle IGNORE_UNKNOWN makes the decompiler work in advanced way, ignoring missing opcodes, incorrect parameters and so on. It may help you It allows to decompile almost every file to crack the protected files (or the files which were compiled incorrectly).
toggle CONSTANT_INDEXES allows to decompile the arrays with the constant number index, if possible. It's available for San Andreas (enabled by default), LCS, VCS. For example,

toggle SKIP_SCM_HEADER

Makes the decompiler skip the scm header while processing. Allows to decompile the external scms from the script.img separately.

Also makes the compiler not to compile the scm header.

Usage of the \debug parameter is an alternate way to switch the advanced options of Sanny Builder. Run Sanny with the parameter \debug=X, where X is a sequence of characters '0' and '1' (without quotes). The characters mean which debug option is enabled/disabled.

Example: \debug=11000

First digit 1 will enable "code_offsets" option, second digit will enable "ignore_unknown" mode. The '0's digits will disable all remaining options.
